## アプリ名の由来

**Stashpin**（スタッシュピン）

- Stash（リスがどんぐりを隠し貯める習性）× Pin（地図のピン留め）
- 「行きたい場所をこっそり貯めておく」イメージ
- テーマ：ピン、記録、行きたい、共有、きっかけSNSの記録

## 1. プロダクト概要

### 解決する課題

SNSで気になるお店・スポットを見つけても、GoogleMapに移動して検索してリストに登録して……という面倒なステップが多い。「どのSNSで見たか」も忘れる。

### ターゲットユーザー

他人と共有もできる。
将来的に友人カップルに共有することも想定。

### コアバリュー

- SNS共有ボタン → 3タップで保存完了
- きっかけのSNS投稿にいつでも戻れる
- ふたりのリストを一緒に育てる体験

### 競合との差別化

- **Go There**（最も近い競合）: 個人向け・機能多め・ビジネス寄りUI
- **本アプリ**: カップル2人専用・かわいいUI・きっかけSNSへの導線・思い出としての記録

## 2. 機能要件

### Phase 1（MVP）- 目標: 1.5〜2ヶ月

#### 2.1 認証

- [ ] Appleログイン（Sign in with Apple）
- [ ] Googleログイン
- [ ] ペアリング機能（招待コードで2人を紐づける）

#### 2.2 スポット保存（コア機能）

- [ ] **Share Extension**: iOSの共有シートからアプリを選択して保存
  - Instagram / TikTok / X / WebページのURLを自動取得
  - App Groupsでメインアプリにデータを渡す
- [ ] 保存シート（Share Extension UI）
  - SNS元URL（自動セット・編集不可）
  - スポット名（手入力）
  - カテゴリ選択（カフェ / ごはん / スイーツ / おでかけ / その他）
  - メモ（任意）
- [ ] アプリ内からの手動追加（+ボタン）

#### 2.3 リスト表示

- [ ] スポット一覧（追加日降順）
- [ ] 誰が追加したか表示（自分 / パートナー）
- [ ] SNS元のアイコン表示（Instagram / TikTok / X / Web）
- [ ] ステータス管理（「行きたい」「行った！」）
- [ ] カテゴリフィルター

#### 2.4 スポット詳細

- [ ] スポット名・カテゴリ・メモ
- [ ] きっかけのSNS投稿リンク（タップで元投稿を開く）
- [ ] 追加日・追加者
- [ ] ステータス変更

#### 2.5 共有・リアルタイム同期

- [ ] Supabase Realtimeでふたりのリストをリアルタイム同期
- [ ] パートナーが追加したらプッシュ通知

## 3. 非機能要件

### パフォーマンス

- リスト表示: 1秒以内
- Share Extensionからの保存: 3秒以内

### セキュリティ

- データはペアリングした2人のみアクセス可能
- Supabase Row Level Security（RLS）で制御

### オフライン

- Phase 1はオンライン必須で問題なし
- Phase 3以降でオフライン閲覧を検討

## 4. 技術スタック

| レイヤー         | 採用技術                       | 備考                         |

| フロントエンド   | SwiftUI                        | iOS 17以上を対象             |

| プッシュ通知     | APNs + Supabase Edge Functions |                              |

| CI/CD | Xcode Cloud または GitHub Actions | |

## 5. データモデル（初期案）

### users

```
id          uuid PK
name        text
avatar_url  text
created_at  timestamp
```

### pairs

```
id          uuid PK
user1_id    uuid FK -> users.id
user2_id    uuid FK -> users.id
created_at  timestamp
```

### spots

```
id            uuid PK
pair_id       uuid FK -> pairs.id
added_by      uuid FK -> users.id
name          text
category      text   -- cafe / food / sweets / outing / other
memo          text
source_url    text   -- きっかけのSNS URL
source_type   text   -- instagram / tiktok / twitter / web
status        text   -- want / visited
visited_at    timestamp
created_at    timestamp
updated_at    timestamp
```

### photos（Phase 2）

```
id          uuid PK
spot_id     uuid FK -> spots.id
storage_url text
created_at  timestamp
```
